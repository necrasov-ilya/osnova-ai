# Черновик архитектуры

Пока архитектура описывается на уровне контуров. Детали появятся после выбора Android-стека, проверки камеры и первых экспериментов с OCR/VLM.

## Основные блоки

- Camera layer - получает поток камеры и отдаёт кадры в sampler.
- Frame sampler - выбирает кадры для быстрой проверки.
- Scene router - определяет тип содержимого.
- Stability gate - отсекает шум, смазанные кадры и повторы.
- Processing queue - хранит визуальные задачи обработки.
- Route workers - выполняют OCR, VLM, LaTeX или смешанную обработку.
- Draft builder - превращает сырой результат в черновик конспекта.
- Note state engine - хранит состояние заметки и место пользователя.
- Smart insert - выбирает место и форму вставки.
- Editor UI - показывает заметку, блоки, очередь и действия пользователя.

## Принцип активации

Camera layer живёт только в активном камерном контейнере. На главном экране и в обычном режиме редактирования заметки не должно быть фонового анализа кадров.

Тяжёлые маршруты получают не поток кадров, а события: стабильный текстовый регион, устойчивый слайд, возможная формула, новый фрагмент доски. Это защищает батарею и делает поведение понятнее.

## Рабочее правило

Архитектура должна обслуживать сценарий заметок. Если модуль удобно нарисован на схеме, но не помогает пользователю быстрее получить проверяемый конспект, его надо пересмотреть.
