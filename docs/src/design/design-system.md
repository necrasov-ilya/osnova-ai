# Дизайн-система

Всё, что связано с палитрой, типографикой, отступами, компонентами, состояниями и анимациями, ведём как дизайн-систему OSNOVA AI.

Material можно использовать как базу для Android-паттернов. При этом OSNOVA AI не должен выглядеть как безликое Material-приложение. Свои компоненты допустимы и ожидаемы, особенно там, где они помогают сценарию камеры, заметки и очереди обработки.

## Зоны ответственности

- Палитра и темы.
- Типографика редактора.
- Компоненты заметки.
- Компоненты очереди обработки.
- Markdown-блоки и preview.
- Raw draft card.
- Состояния блоков: генерация, готово, изменено, fallback.
- Камерный слой и подсказки поверх изображения.
- Форматирующая пилюля в edit sheet.
- Анимации появления, схлопывания и подтверждения.

## Базовая палитра

Эти цвета считаются системными для текущего MVP:

- Акцент: `#AC2954`.
- Дневной фон: `#F5F2EB`.
- Тёмная тема: `#141413`.

Акцент общий для обеих тем. Он используется для главных действий, готовых блоков очереди и выделений. Дневная тема строится вокруг тёплого бумажного фона, тёмная тема вокруг почти чёрного фона без синего уклона.

Системные панели Android, включая статусбар и нижнюю навигацию, должны брать фон активной темы. Интерфейс не должен выглядеть так, будто приложение и оболочка телефона живут в разных палитрах.

## Правило изменений

Если меняется цвет, кнопка, карточка, состояние блока или поведение редактора, это не просто визуальная правка. Решение надо отразить здесь или в дочерней странице дизайн-системы.

## Компоненты Live Camera MVP

Markdown-блок заметки - основной атом редактора. Он хранит `markdown`, `rawSource`, статус и время обновления. В обычном состоянии блок рендерится через Markwon в `TextView`, по тапу открывается edit sheet с исходным Markdown.

Состояния блока:

| Статус | Визуальное поведение |
| --- | --- |
| `generating` | Тонкая акцентная рамка, живой Markdown-preview, кнопка `стоп`. |
| `ready` | Спокойная карточка без лишнего шума. |
| `edited` | Тот же preview, но с пометкой `изменено`. |
| `failed` | Fallback-блок из очищенного OCR, без красной паники. |

Raw draft card живёт рядом с камерой. Она показывает временный источник для Gemma, а не финальный текст заметки. Когда Markdown начинает приходить чанками, raw card визуально сокращается. Это чистый UI-эффект, генерация от него не зависит.

Processing queue показывает компактные пилюли: `Кадр`, `RAW`, `Markdown`, `Блок`, `Fallback`. Очередь нужна не для логов, а для ощущения контроля: пользователь понимает, что приложение видит, ждёт или уже вставило.

Edit sheet открывает исходный Markdown. Над полем находится форматирующая пилюля:

- `B` - bold.
- `I` - italic.
- `H1` - заголовок.
- `•` - список.
- `”` - цитата.
- `` ` `` - inline code.

Позже эту пилюлю надо заменить на полноценный кастомный selection toolbar, который появляется при выделении текста. Для MVP допустима постоянная панель в edit sheet, потому что она уже задаёт поведение Markdown-first редактора.
