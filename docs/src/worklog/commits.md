# Правила коммитов

Коммиты пишем на русском языке, но сохраняем conventional-маску.

```text
feat(scope):
- Добавлен ...
- Реализован ...
- Изменён ...
```

## Формы слов

Используем форму результата:

- Сделан
- Реализован
- Добавлен
- Изменён
- Исправлен
- Обновлён
- Пересобран
- Удалён
- Подключён
- Описан
- Настроен

Не используем инфинитивы вроде "Собрать", "Добавить", "Исправить", "Пересобрать" в пунктах тела коммита.

## Типы

- `feat` - новая функция или заметная возможность.
- `fix` - исправление поведения.
- `docs` - документация.
- `style` - визуальные изменения без смены логики.
- `refactor` - переработка кода без нового поведения.
- `test` - тесты.
- `chore` - обслуживание проекта.
- `build` - сборка, зависимости, конфиги.
- `ci` - автоматизация и проверки.

## Scope

Подходящие scope для старта:

- `docs`
- `pipeline`
- `camera`
- `notes`
- `editor`
- `design-system`
- `android`
- `ocr`
- `vlm`
- `llm`
- `queue`

## Пример

```text
docs(pipeline):
- Описан черновой live camera pipeline
- Добавлены правила маршрутизации OCR, VLM и LLM
- Зафиксированы этапы очереди обработки
```
