# Журнал идей и решений

Здесь фиксируем мысли из обсуждений, промежуточные решения и вопросы для проверки.

## 2026-05-31

- Зафиксирована идея OSNOVA AI как Android-приложения для live-генерации конспектов из камеры.
- Принят mdBook как стартовый формат проектной документации.
- Черновой pipeline вынесен в отдельную страницу.
- Договорились вести русскоязычные коммиты в форме результата: добавлен, реализован, изменён, исправлен.
- Дизайн-система рассматривается как полноценная часть проекта, а не набор разовых визуальных правок.
- Добавлен сценарий пары: главный экран заметок, создание заметки через шторку, экран редактора, раскрытие камеры по кнопке из нижней пилюли.
- Зафиксировано правило: камера и анализ кадров запускаются только после раскрытия камерного контейнера.
- Добавлена идея визуального закрытия камеры: контейнер схлопывает накопленную информацию как анимацию, но не запускает генерацию сам по себе.
- Зафиксирована потребность в трёх режимах: энергосбережение, сбалансированно, производительность.
- Сформирована первая карта технологий: CameraX, ML Kit, stability gate, OCR, VLM/LLM, Note State Engine и Smart Insert.
- Выбран основной on-device runtime для исследования: LiteRT-LM.
- Основной модельный кандидат: Gemma E2B в формате `.litertlm`; сначала смотреть на Gemma4-E2B, fallback - Gemma-3n-E2B.
- Bonsai оставлен как исследовательская text-only ветка, но не как основной VLM для камеры.
